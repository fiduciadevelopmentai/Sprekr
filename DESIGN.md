# Design system

Sprekr should feel private, capable, calm, and human—not like an AI dashboard. It uses original branding, copy, layout, and motion.

## Tokens

| Token | Light | Dark |
| --- | --- | --- |
| `canvas` | calm warm ivory | deep green-charcoal |
| `surface` | soft ivory | raised graphite |
| `text.primary` | graphite | broken white |
| `text.secondary` | muted graphite | softened white |
| `accent` | restrained eucalyptus/teal | adjusted eucalyptus/teal |
| `border` | low-contrast warm gray | low-contrast light gray |

Spacing uses a 4 pt scale. Small controls use a 10 pt radius, compact panels 14 pt, large surfaces 18 pt, and the Flow Bar is fully pill-shaped. Borders are subtle; shadows are rare and quiet. The main app canvas is one calm solid field; navigation gets only enough contrast to preserve hierarchy. Avoid nested cards, neon, decorative gradients, grain, and generic dashboard grids.

## Typography and icons

Onest is the functional interface family for body copy, controls, navigation, labels, and compact status text. Regular, Medium, and Bold are bundled locally as native OpenType resources; the semantic system uses 400/500/600/700 roles, with Core Text synthesizing the intermediate semibold role from those supplied masters. Body copy is 16 pt with approximately 1.7 line height, compact UI copy is 14–15 pt, labels are 11–12 pt semibold uppercase with 0.10–0.14 em tracking, and buttons use 14 pt semibold.

Crimson Text Regular supplies the editorial contrast for page, onboarding, dialog, empty-state, and primary metric headings. Main app headings sit between 34 and 46 pt with 1.0–1.2 line height and −0.02 to −0.03 em tracking. The native app bundles the desktop `.otf`/`.ttf` counterparts rather than web-only `.woff2` files, so every user sees the exact typography without installing fonts. Both license texts ship inside the app and are listed in [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md).

Primary sidebar and dictation symbols use eight locally subsetted Lucide 1.24 glyphs at a consistent 20 pt optical size. They remain decorative inside fully labelled buttons, so VoiceOver receives the action name rather than a private-use Unicode character. The ISC license is bundled and acknowledged in [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md).

## Screens and behavior

- **Onboarding:** focused, keyboard-operable steps with one clear action, visible recovery paths for denied permissions, and an integrated test dictation.
- **Home:** editorial transcript timeline with live Amsterdam-local `Today`, `Yesterday`, and full-date sections. The labels refresh automatically across midnight, each day forms one calm rounded list surface, and search expands inside the first section instead of occupying the native macOS title bar. The search control has stable view identity outside the changing result groups, preserving native focus and Command-A/C/V/Delete even when filtering changes the first day or returns no result. Copy uses a short “Copied to clipboard.” toast.
- **Insights:** quiet, factual metrics and calendar activity; never infer other app content.
- **Dictionary:** a compact local library for unfamiliar names, brands, and uncommon spellings plus explicit **Saved corrections**. Common recognized vocabulary is discarded before it reaches the published word collection, so long histories cannot grow an unnecessary wall of ordinary words. Editing an uncommon word stores its heard form as an alias for the preferred spelling and confirms that future dictations will use the corrected form. Exact words and phrases restore canonical capitals, spaces, and diacritics independently of Smart formatting; only a unique, tightly bounded near-spelling of a previously learned single-word alias may match automatically. Ambiguous forms remain visible for explicit teaching, renamed preferred forms become aliases, and names or brands default to both languages. Search stays inside the rounded workspace so the native title bar remains reserved for traffic lights, window dragging, and the persistent sidebar control; its clear action explicitly restores the text field as first responder after the conditional clear button disappears.
- **Settings:** a dedicated secondary navigation separates General, System, Privacy, and About. General presents Hold and Toggle as two calm rows with independent key chips and progressive Edit controls; no preset grid competes with the user's own choice. The microphone group names the live System Default route, refreshes on hardware changes, and pairs selection with a restrained eight-bar test meter. Writing keeps output language, formatting, and clearly bounded local correction learning together, while Appearance remains separate. System contains lifecycle, sound, permissions, model, and updates; Privacy contains encrypted local storage, export, and confirmed destructive actions. About presents bundled acknowledgements in a native in-app sheet rather than depending on another app to open a Markdown file. Appearance is application-wide: Light and Dark are fixed overrides, while System clears the override and follows macOS immediately. Cloud, billing, team, and account surfaces are deliberately absent because the local app does not implement them.
- **Onboarding talk controls:** Hold and Toggle are configured directly as two independent fields that accept a keyboard key, key combination, or mouse side button. Preset cards and a separate mode choice are intentionally absent, so the owner chooses both controls once and can test them immediately. Global talk actions pass through without firing while either recorder owns the next keyboard or mouse event. An overlapping candidate is rejected before persistence, the previous key remains intact, and an inline message names the control already using it. Accessibility supplies the event-listening access already needed for text delivery; the Talk key step proves that a real system-wide event tap is active instead of requiring a redundant Input Monitoring permission. `Continue` remains visibly muted until both controls are valid and globally registered.
- **Onboarding finish:** the one signature onboarding motion is reserved for the handoff into the product. A six-second native SwiftUI sequence combines a restrained Crimson Text shimmer, compact metaballs, real progress, and clickable privacy and usage tips. Completion crossfades into a drawn check before the Home action appears. Reduce Motion keeps the same information hierarchy while removing travel, rotation, blur, and orbital movement.
- **Direct number, symbol, and email input:** spoken Dutch and English cardinal numbers become locale-appropriate digits before translation. Compounds, negatives, decimals, currencies, percentages, and quantities are supported; grouping starts at five digits and follows the final output language. Ambiguous `een`, dates, times, telephone numbers, codes, versions, IP addresses, ordinals, and overflow remain unchanged. Explicit names for common keyboard, currency, punctuation, quote, and bracket characters become their real symbols. A strong at-sign cue additionally opens a bounded email mode: nearby name words are joined, explicit dot and separator commands are retained, only safe provider/TLD mistakes are repaired, the result is validated and lowercased, and learned Dictionary aliases retain personal spellings. This direct dictation layer remains active when Smart formatting is off, uses URL-, expression-, and typography-aware spacing, and protects ordinary meanings such as “Punt 1,” “het percentage,” social `@handles`, and “een streepje voor.”
- **Smart formatting:** explicit “alinea 1 / alinea 2,” “nieuwe alinea,” and equivalent English cues create real paragraph breaks without leaving command text behind. Sequential “Punt 1 / Punt 2” labels become separate labelled paragraphs, even when punctuation is incomplete, while single references and invalid sequences stay untouched. “Ten eerste / ten tweede” and equivalent English sequences keep their visible labels and become individual paragraphs with a blank line between them, never bullets. A clear “de volgende punten / the following topics” intent phrase can introduce round `•` bullets when at least three short parallel items follow; those items also receive a blank line, while ordinary comma series remain natural prose. Short, direct meaning questions receive typographic quotation marks around the intended term or phrase, while contextual questions such as “Wat betekent dit voor …?” remain natural. Fixed descriptive lead-ins can introduce their continuation with a colon. Medium and long dictations break at clear discourse transitions from about 40 words, cohesive topic shifts from 70 words, or the safest sentence boundary near 70 words after a block reaches 85 words. A final stutter-friendly rough-copy pass removes repeated sound fragments, words, and short exact phrases even when the recognizer inserted commas or sentence stops. It can reconnect a short question split at the same boundary word, while protecting common deliberate emphasis, grammatical doubles, acronyms/proper names, and a normal next sentence with its own finite verb. Common mixed-language spellings, the canonical `Sprekr` form, and unmistakably broken repeated tails receive additional narrow local corrections. The existing Settings toggle controls these intelligent rewriting passes, but not direct number or symbol input.
- **Sidebar:** a 228 pt navigation surface that starts as a persistent 72 pt icon rail whenever the app window opens. Collapse/expand stays in the top chrome, and hovering a compact destination reveals an immediate high-contrast label beside the icon. The former bottom dictation action is now a quiet question-mark Info control. It opens an original 410 pt in-window callout outside the sidebar's layout calculation, keeping the complete surface on screen without shifting navigation. A custom left arrow is bottom-aligned to the Info button's vertical centre in both sidebar states. The flat editorial layout introduces Fiducia Development, the free project, local processing, encrypted storage, and user control without duplicating Settings or adding marketing clutter.
- **Main window:** opens at roughly 88% of the active display's usable width and cannot shrink below 70%. A native delegate clamp backs up AppKit's minimum-size hint because SwiftUI Auto Layout can otherwise allow half-screen or stale saved frames to become too narrow.
- **Window shell:** the native title bar is transparent and separator-free. Its traffic lights sit inside the same subtly eucalyptus-tinted shell as the sidebar: near-white light grey in Light, soft graphite rather than black in Dark. The solid workspace begins below it as one inset 22 pt continuous-corner surface. Collapse/expand lives in the top chrome so it remains available in both sidebar states.
- **Scrolling:** Home, Insights, Dictionary, and Settings keep native trackpad, mouse-wheel, and keyboard scrolling but suppress visible scroll indicators so no system scrollbar interrupts the rounded workspace.
- **Flow Bar:** an original non-activating bottom-screen pill. Idle stays a 52×20 handle and rests four points closer to the bottom edge. Hover reveals one restrained 114×36 control pair: the current `AUTO`/`NL`/`EN` output language and a central microphone action; no scratchpad or unrelated destination is added. The visible type, chevron, and microphone are optically smaller while their full hit areas remain unchanged. Expansion is immediate, while exit uses 160 ms of hysteresis before checking the global pointer against the actual expanded panel frame. This absorbs the synthetic exit events AppKit emits while resizing a non-activating panel and prevents flicker without polling. Clicking the language chip opens an explicit native menu, while genuinely leaving the bar returns it to the handle with a short ease-out transition. Listening uses an 86×25 brand-and-waveform state; transcription switches atomically to a recognizable 52×25 capsule containing only the centered dot-matrix loader for at least 700 ms. Empty recognition follows that same minimum before `No speech detected` appears. Every semantic state change disables implicit SwiftUI animation and applies state, NSPanel frame, hosting frame, and bounds together; outgoing content is never retained inside the next state’s smaller geometry. Motion exists only inside stable capsules—the waveform, loader, countdowns, pressed feedback, and explicit idle-hover interaction—so every captured frame is a complete valid state. Error and clipboard-recovery messages use one 40 pt-high capsule whose width is measured from its text, icon, 8 pt gap, and 12 pt side insets, clamped between 148 and 360 pt. Their shared 3.8-second deadline drives both automatic dismissal and a visible three-point countdown bar. Processing ignores additional toggle input, and successful insertion returns directly to the quiet idle handle before the preloaded completion sound starts after a 70 ms render margin. Stopping and clipboard recovery are silent; only direct successful delivery owns the completion cue. Escape preserves the audio for six seconds in a 200×40 `Recording cancelled` state with a clear **Undo** action and countdown. Undo transcribes the retained recording, prefers a newly focused editable field, and otherwise copies the restored text with explicit paste guidance.
- **Output language:** `Automatic` keeps the detected spoken language. Selecting `Nederlands` or `English` makes that language the final inserted text: same-language speech remains untouched, while the opposite language is translated locally with Apple's Translation framework after recognition and smart formatting. Valid email addresses are temporarily protected as exact local literals so translation changes only the surrounding prose. Translation or literal-restoration failure never discards the transcript; the source text is saved and delivered with a clear recovery message.
- **Brand assets:** the supplied transparent seven-bar mark replaces the generic waveform in the in-app brand lockup and listening state. The Dock icon keeps its rounded near-black container and seven-bar artwork, while the four pixels regions outside that rounded shape use real alpha rather than a black square backdrop. Every macOS icon size is regenerated from the editable SVG and the build rejects a master PNG without transparency. A custom 420 ms softly faded two-tone completion sound replaces the brighter system `Glass` alert. Recording start remains restrained, stopping is silent, and successful delivery owns the single completion cue.

## Motion and accessibility

- Interactive surfaces respond immediately: sidebar destinations accept the first click even from an inactive window, hover states tint the whole hit target, and pressed states deepen briefly.
- Hover and pressed colors remain visible in both Light and Dark appearances. Reduce Motion removes scale interpolation while preserving immediate state-color feedback.

Normal state changes use 180–260 ms natural ease-out transitions; onboarding moves use 280–360 ms. The audio meter uses a quick attack and slower release so speech feels responsive without jitter. Motion communicates state and direction, uses opacity/transforms, avoids bounce, and fully respects Reduce Motion: the dot matrix becomes an opacity pulse, and the Undo countdown updates once per second instead of animating at 30 fps. All interactive controls support keyboard navigation, VoiceOver labels, sufficient contrast, and practical larger-text layouts.
